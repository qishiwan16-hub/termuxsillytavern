更新时间：2026-03-05

## 1. 项目一句话
这是一个运行在 `127.0.0.1` 的本地 WebApp，用于在不打开 SillyTavern 页面时，管理 SillyTavern 资源与 Vault 素材库，支持导入导出、文件编辑、Git、队列、回收站、认证等能力。

## 3. 各顶层目录详解

## 3.1 `src/`（前端源码，14 个文件）
前端 React + TypeScript 代码。

包含子目录：

1. `src/app/`
2. `src/lib/`
3. `src/styles/`

包含顶层文件：

1. `src/main.tsx`
2. `src/styles.css`
3. `src/vite-env.d.ts`
4. `src/LegacyApp.tsx`

用途说明：

1. `main.tsx`：前端入口，挂载 `<App />`，并做“启动崩溃兜底页”（白屏时显示错误堆栈）。
2. `LegacyApp.tsx`：旧版页面（并行兼容入口）。
3. `styles.css`：主样式入口，转发导入 `styles/mobile.css` 与 `styles/legacy.css`。
4. `vite-env.d.ts`：Vite 类型声明。

### 3.1.1 `src/app/`（新 UI 主应用）
包含文件：

1. `src/app/App.tsx`
2. `src/app/types.ts`
3. `src/app/ui-config.ts`
4. `src/app/components/AuthGate.tsx`
5. `src/app/components/HomePage.tsx`
6. `src/app/components/ProfileEditorModal.tsx`
7. `src/app/components/panels/MainPanels.tsx`

用途说明：

1. `App.tsx`：页面主状态机与业务协调中心。负责认证流程、实例加载、资源加载、批量操作、云端/Git 操作、路径设置、收藏路径、目录浏览器、弹层开关等。
2. `types.ts`：前端类型定义（Dashboard/Queue/Resource/Settings 等）。
3. `ui-config.ts`：UI 常量与工具函数（本地存储键、首页显示标签、日期格式、首字母等）。
4. `AuthGate.tsx`：登录/首次设置密码界面。
5. `HomePage.tsx`：首页卡片与入口按钮（资源中心、分类、系统记录、云端入口等）。
6. `ProfileEditorModal.tsx`：个人资料与系统设置弹窗（头像、用户名、字体、项目路径、路径收藏、认证开关、改密）。
7. `MainPanels.tsx`：资源/队列/Git/云端/设置五类面板组件合集。

### 3.1.2 `src/lib/`
包含文件：

1. `src/lib/api.ts`

用途说明：

1. 统一封装 API 请求、Bearer Token 注入、错误处理、ZIP 下载行为。

### 3.1.3 `src/styles/`
包含文件：

1. `src/styles/mobile.css`
2. `src/styles/legacy.css`

用途说明：

1. `mobile.css`：当前主视觉样式与移动端适配（首页、弹窗、卡片、面板等）。
2. `legacy.css`：旧版兼容样式。

## 3.2 `server/`（后端源码，24 个文件）
Fastify + TypeScript 后端，负责所有文件系统/Git/队列/Vault/安全策略/API。

包含子目录：

1. `server/lib/`
2. `server/tests/`

包含顶层文件：

1. `server/index.ts`
2. `server/types.ts`

用途说明：

1. `index.ts`：后端主入口，注册 API 路由与中间件，加载服务实例，处理认证、目录浏览、资源、回收站、Git、导入导出、文件编辑、设置等。
2. `types.ts`：后端统一类型定义（Instance、VaultItem、DashboardSummary、TrashItem、AppSettings 等）。

### 3.2.1 `server/lib/`（核心服务层）
包含文件及职责：

1. `env.ts`：定义数据目录路径（`~/.st-resource-manager`）、默认实例路径、目录初始化。
2. `instance-store.ts`：实例配置管理（增删改查、布局类型检测、运行状态刷新）。
3. `vault-service.ts`：Vault 元数据与文件管理（导入、检索、标签、收藏、取用、Git 目录协同）。
4. `queue-service.ts`：写入队列管理（入队、状态流转、阻塞/执行/持久化）。
5. `scan-cache-service.ts`：资源扫描缓存与增量刷新。
6. `resource-center-service.ts`：统一资源列表聚合（实例 + Vault）、筛选分页、预览能力信息。
7. `dashboard-service.ts`：首页汇总数据构建（实例状态、队列统计、资源统计、版本探测）。
8. `trash-service.ts`：软删除回收站（移入、恢复、永久删除、过期清理）。
9. `file-tree.ts`：目录树与资源扫描（分页、过滤、类型统计）。
10. `resource-type.ts`：通过路径/扩展名推断资源类型。
11. `file-service.ts`：文件读取与写入（含大文件策略、备份前写入）。
12. `backup.ts`：备份创建、轮转保留、恢复。
13. `path-safety.ts`：路径安全校验（防越界、防穿越、相对路径清洗）。
14. `zip-service.ts`：实例 ZIP 导入导出。
15. `mixed-zip-service.ts`：混合来源资源打包（实例+Vault 批量导出）。
16. `git-service.ts`：Git clone/commit/pull/push 封装。
17. `auth-service.ts`：认证开关、密码与 token 管理。
18. `app-settings-service.ts`：应用设置存取（保留天数、自动选项等）。
19. `runtime.ts`：运行状态检测（SillyTavern/实例运行标记）。
20. `upload.ts`：multipart 上传文件读取。
21. `audit.ts`：审计日志追加写入。

### 3.2.2 `server/tests/`
包含文件：

1. `server/tests/run-tests.ts`

用途说明：

1. 核心后端能力验证脚本（路径安全、队列、备份轮转、Vault 搜索、认证生命周期、扫描分页、回收站生命周期）。

## 3.3 `scripts/`（运维脚本，5 个文件）
部署、启动、构建、Termux 一键流程。

包含文件及职责：

1. `scripts/termux-oneclick.sh`：主一键脚本。支持 `start/stop/restart/status/logs/init-dirs/config`，包含自动更新、依赖检查、后台启动、健康检查等。
2. `scripts/install-termux.sh`：手动初始化脚本（依赖检测、安装、构建、环境信息记录）。
3. `scripts/start.sh`：标准启动脚本，设置 `HOST/PORT` 后执行 `npm run start`。
4. `scripts/start-quick.sh`：仅依赖运行时环境快速启动（要求已有 `dist`）。
5. `scripts/build-client.mjs`：前端构建脚本，使用 esbuild 输出 `dist/client`，避免某些移动端/模拟器构建兼容问题。

## 3.4 `dist/`（构建产物，28 个文件）
打包后的可运行文件，不建议手工改。

包含子目录：

1. `dist/client/`：前端静态资源（`index.html` + `assets/index.js/index.css`）。
2. `dist/server/`：后端 JS 编译输出（含 `lib/`、`tests/`）。

说明：

1. 正式启动时读取 `dist/server/index.js`。
2. 一键脚本会在必要时重新构建此目录。

## 3.5 `node_modules/`（依赖目录，约 9425 个文件）

## 5. 运行与构建流（你可以这样理解）

1. 开发模式：
`npm run dev` 启动前后端并发进程（Vite + tsx watch）。
2. 构建模式：
`npm run build` 先构建前端到 `dist/client`，再编译后端到 `dist/server`。
3. 运行模式：
`npm run start` 运行 `dist/server/index.js`，后端提供 API 与静态页面。
4. Termux 一键模式：
`termux-oneclick.sh`/`scripts/termux-oneclick.sh` 负责更新仓库、检查依赖、构建、后台启动、状态和日志。

## 6. 建议的阅读顺序（初学者友好）

1. 先看 `package.json`：知道命令入口。
2. 再看 `server/index.ts`：知道后端 API 全景。
3. 看 `src/app/App.tsx`：知道前端状态与业务主流程。
4. 看 `server/lib/`：按模块理解扫描、队列、Vault、回收站、Git、安全。
5. 最后看 `scripts/termux-oneclick.sh`：理解部署与运行自动化。
