# 全局 UI 风格规范 v1

## 1. 目标
- 以你提供的参考图为唯一视觉锚点，统一全站为高相似度的轻灰极简风。
- 优先保证版式、字重、留白、信息节奏一致。
- 去除不必要边框，仅在信息分组与可点击区域保留弱化分隔。

## 2. 视觉基因
- 底色：浅灰背景，接近纸感。
- 主字：深灰到近黑，强调标题的衬线气质。
- 次字：中灰，承担说明与标签。
- 点缀：极少量状态色，仅用于提示点与关键状态。
- 组件轮廓：默认无强边框，采用微弱层级与留白建立结构。

## 3. 设计 Token
### 3.1 颜色
- 页面底色：#f2f2f2
- 面板底色：#f5f5f5
- 一级文字：#161616
- 二级文字：#6f6f6f
- 三级文字：#a0a0a0
- 弱分隔线：#e9e9e9
- 焦点深色：#2a2a2a
- 状态点绿：#22c55e

### 3.2 字体体系
- 标题字体：Noto Serif SC
- 正文字体：Noto Sans SC
- 页面主标题：48-60
- 模块标题：28-36
- 卡片标题：42-52
- 正文：26-30
- 标签与辅助：20-24

### 3.3 间距与栅格
- 外边距基线：24
- 区块间距：20 或 24
- 卡片内边距：20
- 卡片圆角：0 或 8，仅在必要场景使用
- 栅格：手机双列卡片采用 1fr 1fr，间距 12

### 3.4 边框与阴影
- 默认不加边框。
- 仅以下场景允许边框：
  - 大区块分组容器
  - 表单输入焦点态
  - 高价值可点击项的 hover 或 active 态
- 线宽统一 1。
- 阴影禁用，改用明度差区分层级。

## 4. 页面结构规范
### 4.1 顶部导航条
- 左侧返回文字，中央或左对齐大标题，右侧刷新图标。
- 顶部区域保持大留白，增强呼吸感。

### 4.2 列表页模板
- 卡片纵向堆叠。
- 每卡结构：
  - 顶部微标签
  - 主标题
  - 副标题
  - 底部动作文案
- 不使用粗边框和重背景块。

### 4.3 详情页模板
- 头部：返回 + 标题 + 头像。
- 横幅区：图片优先，无图占位。
- 内容区：2 列信息卡 + 1 行横向控制卡 + 2 列补充卡。
- 保持信息密度低，避免一次塞满字段。

### 4.4 表单页模板
- 输入项以垂直流布局。
- 每项含标签、输入区、可选说明。
- 提交按钮采用单主按钮，其他操作弱化。

## 5. 组件级规则
### 5.1 按钮
- 常规按钮：浅底深字，无厚边。
- 主按钮：深底浅字。
- 文字按钮：无边框，仅文字。

### 5.2 卡片
- 首选无描边卡片。
- 必须分组时使用极弱线条。
- 选中态优先通过背景明度变化，不使用重描边。

### 5.3 图标与状态点
- 图标线性化、低对比。
- 状态点只保留一个高亮色，禁止多色竞争。

## 6. 现有项目映射策略
- 基线样式源于 [`src/styles/mobile.css`](../src/styles/mobile.css)。
- 页面结构入口位于 [`src/app/components/HomePage.tsx`](../src/app/components/HomePage.tsx)。
- 面板样式与内容位于 [`src/app/components/panels/MainPanels.tsx`](../src/app/components/panels/MainPanels.tsx)。
- 全局编排位于 [`src/app/App.tsx`](../src/app/App.tsx)。

## 7. 分阶段落地顺序
1. 第一阶段：角色卡列表页与角色详情页
2. 第二阶段：首页与资源管理总览
3. 第三阶段：预设、云端、队列、Git、系统设置
4. 第四阶段：认证页、弹窗、边界态与空态

## 8. 验收标准
- 视觉相似度：版式、留白、层级与参考图一致。
- 统一性：全站字重、字号、色阶、间距遵循同一 token。
- 边框克制：不存在不必要边框。
- 响应式：主流手机宽度下不破版。
- 可维护：仅通过 `m-*` 样式层迭代，避免散乱内联样式。
